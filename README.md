# MY-3D-portfolio

React, TypeScript, Vite, Three.js 기반으로 제작한 3D 개발 포트폴리오 프로젝트입니다.  
인터랙티브한 히어로 섹션과 모션 연출을 통해, 단순한 소개 페이지가 아니라 **시각적으로 기억에 남는 포트폴리오 경험**을 만드는 것을 목표로 했습니다.

## 프로젝트 소개

이 프로젝트는 웹 포트폴리오를 더 인상적으로 보여주기 위해 시작했습니다.

일반적인 정적 포트폴리오 페이지를 넘어서,

- 첫 화면에서 강한 인상을 주는 3D 히어로 섹션
- 섹션 간 자연스러운 흐름
- 기술 스택, 프로젝트, 소개 영역의 일관된 구성
- 시각 연출과 가독성 사이의 균형

을 중심으로 직접 구조를 다듬고 커스터마이징한 포트폴리오입니다.

## 핵심 포인트

- 3D 기반 히어로 섹션 구성
- React 컴포넌트 중심의 화면 구조
- TypeScript 기반 프로젝트 구성
- Vite 기반 개발 환경
- 포트폴리오용 UI/UX 흐름 정리
- 시각 효과와 실제 콘텐츠 전달의 균형 조정

## 기술 스택

- **Frontend**: React, TypeScript
- **Build Tool**: Vite
- **3D / Motion**: Three.js
- **Styling**: CSS / UI 커스터마이징
- **Etc.**: 포트폴리오 구조 설계, 인터랙션 조정

## 이 프로젝트에서 중요하게 본 점

### 1. 첫 화면의 인상
포트폴리오에서 가장 먼저 보이는 히어로 섹션의 완성도를 중요하게 생각했습니다.  
단순히 3D 요소를 넣는 것이 아니라, 콘텐츠를 방해하지 않으면서도 시선을 끌 수 있도록 조정했습니다.

### 2. 시각 효과와 가독성의 균형
3D 연출이 강해질수록 텍스트 전달력이 약해질 수 있기 때문에,  
배경, 오브젝트, 글자 대비와 섹션 구성의 균형을 계속 수정하며 다듬었습니다.

### 3. 템플릿 사용이 아니라 구조 이해 후 커스터마이징
겉모습만 바꾸는 수준이 아니라,  
컴포넌트 구조와 히어로 섹션 연출 방식을 분석하고 필요한 방향으로 수정하는 데 집중했습니다.

## 내가 집중한 부분

- 히어로 섹션 비주얼 완성도 개선
- 중앙 오브젝트와 배경의 자연스러운 연결감 조정
- 포트폴리오형 섹션 흐름 정리
- 시각 연출이 과하지 않도록 UI 밀도 조절
- 추후 개인 프로젝트 소개용으로 확장 가능한 구조 유지

## 아쉬운 점

- 배포 버전 기준으로 성능 최적화를 더 진행할 수 있습니다.
